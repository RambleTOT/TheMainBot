# Инструкция для заказчика: бот, оплаты (ЮKassa) и хостинг (Beget)

Выполняйте по шагам. Всё, что помечено 📤 — нужно прислать разработчику.
Технической настройкой занимается разработчик; от вас — регистрация, доступы и данные.

Порядок: **1) создать бота → 2) подключить ЮKassa → 3) купить хостинг (Beget)**.

---

# Часть 1. Создание бота в Telegram (BotFather)

Бот создаётся на **вашем** Telegram-аккаунте — так он полностью принадлежит вам.

1. В поиске Telegram найдите **@BotFather** (с синей галочкой — это официальный бот).
2. Откройте его и нажмите **«Запустить» / Start**.
3. Отправьте команду **`/newbot`**.
4. Введите **имя бота** — отображаемое название, любое (например: `THE MAIN`).
5. Введите **username бота** — латиницей, обязательно заканчивается на `bot`
   (например: `themain_access_bot`). Если занято — придумайте другой.
6. BotFather пришлёт сообщение с **токеном** — длинная строка вида
   `123456789:AAE-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`.
   - ⚠️ **Токен = полный доступ к боту.** Никому не показывайте. Если случайно утёк —
     его можно перевыпустить командой `/token`.

📤 **Прислать разработчику:** username бота и токен (токен — **безопасным способом**,
не в общем чате: отдельным личным сообщением или архивом с паролем).

> Аватар, описание и меню команд настроит разработчик — вам об этом думать не нужно.

# Часть 2. Приём оплат — ЮKassa

## 2.0. Важно: не перепутать сервисы
- Нужна именно **ЮKassa** (сайт **yookassa.ru**) — приём платежей для бизнеса
  (карты, СБП, автосписания, чеки 54-ФЗ).
- **ЮMoney** (кошелёк) — это другое, не подойдёт.
- Оформляем на ваше **ИП**.

## 2.1. Что подготовить заранее
- ИНН и ОГРНИП ИП.
- Паспортные данные ИП.
- **Расчётный счёт ИП** в банке — на него будут приходить деньги.
- Система налогообложения (например, «УСН доходы») — понадобится для чеков.
- Телефон и e-mail.

## 2.2. Регистрация в ЮKassa (шаги)
1. Зайдите на **yookassa.ru** → **«Подключить»** (регистрация для бизнеса).
2. Войдите/создайте аккаунт (Яндекс ID / Ю ID).
3. Выберите тип — **ИП**, заполните данные ИП и реквизиты расчётного счёта.
4. Создайте **магазин** (точка приёма оплат). Тип приёма — **«Через API»**
   (для Telegram-бота). Если попросят сайт и описание — см. п. 2.3.
5. Подпишите оферту/договор (электронно) и отправьте на проверку (модерацию).
   Обычно **1–3 рабочих дня**.

> 💡 Пока идёт модерация, разработчик уже начинает интеграцию на **тестовом магазине**
> (тестовые ключи доступны сразу) — время не теряется.

## 2.3. Возможное требование модерации — простой сайт
ЮKassa при проверке часто просит **сайт/страницу** с описанием услуги, **офертой**,
**политикой конфиденциальности** и контактами. У бота своего сайта нет, поэтому:
- либо у вас уже есть лендинг проекта — дайте его адрес;
- либо разработчик сделает **простую страницу-оферту** (1 страница) — сообщите, нужно ли.
- Тексты **оферты** (с условием автосписаний) и **политики** — желательно от юриста.

## 2.4. Настройка чеков (54-ФЗ)
Чеки при оплате от физлиц обязательны. В кабинете ЮKassa:
1. Раздел про **чеки / онлайн-кассу**. Если своей кассы нет — подключите
   **кассовое решение через ЮKassa** (часть — платная, небольшая сумма).
2. Укажите **систему налогообложения** и **ставку НДС** (для УСН обычно «без НДС»).
3. 📤 Пришлите разработчику: какие поставили **систему налогообложения** и **ставку НДС**
   (или дайте доступ, чтобы он настроил чеки в интеграции).

## 2.5. Включить способы оплаты и автосписания
В кабинете ЮKassa включите/проверьте:
- **Банковские карты** — обычно включены по умолчанию.
- **СБП** — при необходимости активируйте отдельно (комиссия ниже, чем по картам).
- **Автоплатежи / сохранение способа оплаты** — нужно для автопродления. Если опции нет —
  напишите в поддержку ЮKassa: «включите сохранение платёжного метода для рекуррентных платежей».

## 2.6. Ключи для разработчика (самое важное)
1. Раздел **«Настройки» → «API-ключи»** (или «Интеграция»).
2. Возьмите:
   - **shopId (идентификатор магазина)** — 📤 прислать;
   - **Секретный ключ (API-ключ)** — создайте и 📤 пришлите **боевой** и **тестовый**.
3. ⚠️ **Секретный ключ = доступ к деньгам.** Пришлите **отдельно и безопасно**
   (личным сообщением / архивом с паролем). Ключ всегда можно перевыпустить.
4. **Webhook (уведомления об оплате):** разработчик даст адрес вида
   `https://ваш-домен/yookassa` — его указывают в кабинете ЮKassa (или дайте доступ разработчику).

### 📤 Итог по ЮKassa
- [ ] shopId (тест и боевой)
- [ ] Секретный API-ключ (тест и боевой) — **безопасно**
- [ ] Параметры чека: система налогообложения + ставка НДС (или доступ в кабинет)
- [ ] Подтверждение: карты, СБП, автоплатежи включены
- [ ] Решение по странице-оферте (есть сайт / делаем страницу)

# Часть 3. Хостинг — Beget

Бот и база данных работают круглосуточно на сервере. Под вашу нагрузку
(**1 000–2 000 пользователей в месяц**) нужен небольшой сервер — это недорого.

## ⚠️ Важно: берём VPS, а НЕ «виртуальный хостинг»
У Beget есть два разных продукта:
- ❌ **«Виртуальный хостинг»** — для обычных сайтов, **боту не подойдёт**.
- ✅ **«VPS» / «Облачные серверы»** — то, что нужно. **Берите именно его.**

## 3.1. Как купить VPS на Beget (шаги)
1. Зайдите на **beget.com** и зарегистрируйтесь **на свои данные** (аккаунт должен быть ваш).
2. В панели откройте раздел **«VPS»** (облачные серверы). *Не «Виртуальный хостинг».*
3. Нажмите **«Создать / Заказать сервер»**.
4. Операционная система — **Ubuntu 24.04**.
5. Конфигурация (тариф):
   - **2 CPU**
   - **4 ГБ RAM**
   - **~30–50 ГБ NVMe/SSD**
6. Включите **резервные копии (бэкапы)**, если есть такая опция.
7. Оплатите (пополните баланс). Стоимость — см. п. 3.3.
8. После создания у сервера появятся **IP-адрес** и данные доступа (root-пароль).

> Этого хватит с большим запасом. Если пользователей станет кратно больше —
> тариф расширяется в пару кликов, без переустановки.

## 3.2. Домен (нужен для безопасной оплаты)
Уведомления об оплате ЮKassa приходят по защищённому адресу (HTTPS), поэтому нужен домен.
Удобно купить его **там же, на Beget** — всё в одной панели:
1. В панели Beget откройте **«Домены»** → зарегистрируйте домен **.ru**
   (например, `themain-bot.ru`) на свои данные.
2. Управление DNS — в той же панели Beget. Разработчику нужен доступ к DNS
   (или вы добавите одну запись по его инструкции). Сертификат HTTPS настроит разработчик (бесплатно).

## 3.3. Примерная стоимость
- VPS (2 CPU / 4 ГБ): **~700–1 000 ₽/мес**.
- Домен .ru: **~200–600 ₽/год**.
- **Итого ≈ до 1 000 ₽/мес** — на фоне проекта незначительно, зато стабильно 24/7.

- [ ] IP-адрес сервера
- [ ] Доступ к серверу: root-пароль **или** добавить SSH-ключ разработчика (он пришлёт ключ)
- [ ] Имя домена + доступ к DNS в панели Beget (или готовность добавить запись по инструкции)

# Часть 4. Кому что принадлежит
Всё регистрируйте **на себя** — тогда проект полностью ваш:
- Бот (@BotFather) — на вашем Telegram-аккаунте; разработчику — только токен.
- Кабинет ЮKassa — на ваше ИП (деньги идут напрямую вам).
- Сервер и домен (Beget) — на ваш аккаунт и вашу оплату.
- Все пароли/ключи — у вас; разработчику — рабочие копии для настройки и поддержки.
