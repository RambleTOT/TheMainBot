# Кастомные эмодзи (стикеры) в тексте и админ-панель

## 1. Кастомные эмодзи в сообщениях бота

### Как это работает
Кастомные (в т.ч. анимированные) эмодзи в Telegram — это по сути мини-стикеры,
которые встраиваются прямо в текст. Их **видят все пользователи**, включая тех, у кого
нет Telegram Premium.

### ⚠️ Главное ограничение (без этого не заработает)
Бот может **отправлять** кастомные эмодзи в своих сообщениях только если выполнено одно из:
- **у владельца бота есть Telegram Premium** (рекомендуемый путь), **или**
- для бота куплен дополнительный username на Fragment (экзотика, не нужно).

То есть заказчику нужно оформить **Telegram Premium на том аккаунте, которому принадлежит бот**.
После этого бот сможет вставлять кастомные эмодзи в приветствие, описание приватки и т.д.
Если Premium у владельца нет — бот сможет показывать только **обычные** эмодзи (как сейчас).

### Как создать свой набор кастомных эмодзи
1. У владельца должен быть Telegram Premium.
2. Открыть бота **@Stickers** → создать **эмодзи-пак** (New emoji pack).
3. Загрузить картинки эмодзи: статичные (PNG/WEBP 100×100) или анимированные (TGS)
   или видео (WEBM), к каждому привязать обычный эмодзи-«дублёр».
4. Готово — у каждого кастомного эмодзи появляется свой `custom_emoji_id`.

> Можно и не создавать свой пак: при Premium у владельца бот может слать эмодзи
> из **любого** набора. Но обычно делают брендовый пак под проект (слон 🐘 и т.п.).

### Как мы вставим их в тексты (со стороны разработки)
Технически кастомный эмодзи — это специальная «разметка» сообщения
(`message entity` типа `custom_emoji` со ссылкой на `custom_emoji_id`).
Чтобы заказчик мог **сам** ставить их из админки, мы добавим простой синтаксис в тексты:

```
🐘 База знаний        ← обычный эмодзи (работает всегда)
[ce:5411885707371940229] База знаний   ← кастомный эмодзи по его id
```

Бот при отправке заменит `[ce:ID]` на нужный кастомный эмодзи. Так владелец сможет
вставлять кастомные эмодзи в любой текст прямо из веб-админки.

### Что нужно от заказчика, чтобы включить кастомные эмодзи
1. Оформить **Telegram Premium** на аккаунте-владельце бота.
2. Прислать **набор кастомных эмодзи** (или ссылку на эмодзи-пак) и список
   `custom_emoji_id` тех эмодзи, которые хотим использовать (поможем их получить).

После этого включение `[ce:...]` в тексты — небольшая доработка (несколько часов).
До этого момента везде используются обычные эмодзи — всё работает.

## 2. Админ-панель (веб)

Отдельная веб-панель, где заказчик **сам меняет тексты и цены** и управляет пользователями.
Данные общие с ботом — правки применяются в боте сразу.

### Что умеет
- **Тексты** — редактирование всех сообщений бота: приветствие, описание приватки,
  тарифный дисклеймер, оферта, тексты «Моя подписка», благодарность, тексты подарков и т.д.
  Поддерживается HTML (`<b>`, `<i>`, `<a href>`).
- **Тарифы и цены** — изменение цен, названий, эмодзи, срока, порядка; включение/выключение
  и добавление/удаление тарифов.
- **Пользователи** — поиск по ID/username, статус подписки, ручная выдача подписки,
  остановка автоплатежа, отмена подписки, блокировка/разблокировка.

### Как запустить (локально)
```bash
# в .env задать ADMIN_PASSWORD и ADMIN_SECRET
uvicorn admin.app:app --host 127.0.0.1 --port 8000
```
Открыть **http://127.0.0.1:8000/admin** , войти по паролю из `ADMIN_PASSWORD`.

### На боевом сервере (Beget)
- Адрес: **https://themainpsychology.ru/admin**
- Панель работает рядом с ботом на том же сервере и той же базе (PostgreSQL).
- Доступ — **только по HTTPS и под паролем** (в идеале ещё ограничить по IP — блок
  `allow/deny` заготовлен в `deploy/nginx-themain.conf`). Настраивается при развёртывании.

### Безопасность
- Вход по паролю (`ADMIN_PASSWORD`), сессия — подписанная cookie (`ADMIN_SECRET`).
- Все действия, меняющие данные, — только после входа.
- Пароль и секрет хранятся в `.env` на сервере, не в коде.
