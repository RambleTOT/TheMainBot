# THE MAIN — Telegram-бот подписок + админка

Бот продаёт подписки на закрытые ресурсы (приватный канал/форум/чат) с тарифами
1 / 3 / 6 / 12 месяцев и «Навсегда», ведёт срок по московскому времени, поддерживает
автопродление и его остановку, подарочные подписки и доступ по заявкам с одобрением ботом.

Все **тексты и цены редактируются из веб-админки** (хранятся в БД).

> **Оплата пока в демо-режиме** (кнопка «✅ Я оплатил (тест)»). Реальная интеграция
> ЮKassa (платёж, вебхук, автосписания, чеки 54-ФЗ) — следующий этап.

## Возможности

- `/start` — приветствие (текст/картинка) + постоянное нижнее меню:
  **💰 Тарифы**, **🔑 Моя подписка**, **⚙️ Подробнее о приватке**.
- Экран тарифов (5 тарифов с эмодзи) и карточка тарифа с дисклеймером и кнопкой оферты.
- Кнопки **💳 Оплатить** и **🎁 Оплатить для друга** (подарок → одноразовая ссылка).
- «Моя подписка»: статус, срок (МСК) или «безлимитный», состояние автоплатежа.
- Остановка автопродления с подтверждением.
- Доступ к каналу через **заявки на вступление** — бот одобряет только оплативших.
- Планировщик: автопродление и отзыв доступа по окончании.
- **Веб-админка**: редактирование текстов, цен/тарифов, управление пользователями.

## Запуск (локально)

```bash
cd /Users/rambletot/Bot
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env      # задать BOT_TOKEN, ADMIN_PASSWORD, ADMIN_SECRET

# бот:
python -m bot.main

# админка (в отдельном терминале, из корня проекта):
uvicorn admin.app:app --host 127.0.0.1 --port 8000
# затем http://127.0.0.1:8000/admin , вход по ADMIN_PASSWORD
# (на боевом сервере — https://<домен>/admin)
```

При первом запуске тексты и тарифы заполняются значениями по умолчанию, дальше
редактируются из админки.

> Dev-примечание: при изменении схемы БД в SQLite нет авто-миграций — просто удалите
> `bot.db`, он пересоздастся. В production — PostgreSQL + миграции.

## Структура

```
bot/
  main.py        — запуск бота, планировщик, сидирование
  config.py      — настройки из .env (токен, БД, пароль админки, ресурсы)
  content.py     — тексты и тарифы (значения по умолчанию + загрузка/сохранение в БД)
  texts.py       — только подписи кнопок (участвуют в маршрутизации)
  keyboards.py   — клавиатуры (нижнее меню + inline)
  database.py    — модели: User, Subscription, Payment, GiftCode, Tariff, Setting
  services.py    — подписки, автопродление, подарки, время МСК
  access.py      — выдача (заявки) и отзыв (кик) доступа
  payments.py    — интерфейс платежей + мок (заготовка под ЮKassa)
  scheduler.py   — задачи по расписанию
  handlers.py    — обработчики команд и кнопок
admin/
  app.py         — FastAPI-админка (тексты, цены, пользователи)
  templates/     — HTML-шаблоны
```
