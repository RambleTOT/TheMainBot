# Подключение к серверу по SSH (для бота THE MAIN)

SSH — это способ зайти в командную строку сервера со своего компьютера.
Ниже — полная инструкция: что нужно, как подключиться с Windows и с Mac/Linux,
первые действия на сервере, переход на ключи и частые проблемы.

---

## 1. Что нужно (взять в панели Beget)
В панели Beget → раздел **VPS / Облачные серверы → ваш сервер → «Доступ» (или «Root»)**:
- **IP-адрес** сервера (например, `85.198.xxx.xxx`);
- **логин** — обычно `root`;
- **пароль** root (или возможность задать/сбросить его там же);
- **порт SSH** — по умолчанию `22` (если Beget указал другой — используйте его).

> ⚠️ Пароль root — это полный доступ к серверу. Не пересылайте его в мессенджерах/чатах.

---

## 2. Подключение с Mac или Linux
Откройте приложение **«Терминал»** и введите (подставьте свой IP):
```bash
ssh root@85.198.xxx.xxx
```
Если порт не 22 — добавьте `-p ПОРТ`:
```bash
ssh -p 22 root@85.198.xxx.xxx
```
- При первом подключении спросит `Are you sure you want to continue connecting?` — введите **yes**.
- Затем запросит пароль — вводите root-пароль (символы не отображаются, это нормально), Enter.
- Готово: вы в командной строке сервера (приглашение вида `root@vps:~#`).

---

## 3. Подключение с Windows
**Вариант А — встроенный SSH (Windows 10/11):**
1. Откройте **PowerShell** (Пуск → наберите «PowerShell»).
2. Команда:
   ```powershell
   ssh root@85.198.xxx.xxx
   ```
3. `yes` на вопрос о ключе, затем пароль.

**Вариант Б — PuTTY (если PowerShell недоступен):**
1. Скачайте PuTTY с putty.org и установите.
2. В поле **Host Name** впишите IP, **Port** — 22, тип — SSH → **Open**.
3. В окне: `login as:` → `root`, затем пароль.

---

## 4. Первые действия после входа (один раз)
```bash
# обновить систему
apt-get update && apt-get upgrade -y

# сменить root-пароль на свой надёжный (если Beget выдал временный)
passwd

# поставить московское время (для логов)
timedatectl set-timezone Europe/Moscow
```

---

## 5. Рекомендуется: вход по SSH-ключу (без пароля, безопаснее)
На **своём компьютере** (Терминал/PowerShell):
```bash
# 1) создать ключ (если ещё нет). Enter на все вопросы:
ssh-keygen -t ed25519

# 2) скопировать ключ на сервер (введёте пароль root последний раз):
ssh-copy-id root@85.198.xxx.xxx
#   На Windows, если ssh-copy-id нет — покажите мне вывод команды:
#   type $env:USERPROFILE\.ssh\id_ed25519.pub
#   и я подскажу, как добавить его на сервер.
```
После этого `ssh root@IP` пускает без пароля.

*(По желанию, позже) отключить вход по паролю совсем — скажу отдельно, это делается
в `/etc/ssh/sshd_config`. Сначала убедитесь, что вход по ключу работает.*

---

## 6. Полезные команды на сервере (для бота)
```bash
# статус сервисов
systemctl status themain-bot
systemctl status themain-admin

# живые логи бота (Ctrl+C — выйти)
journalctl -u themain-bot -f

# перезапустить после изменений
systemctl restart themain-bot themain-admin

# редактировать настройки (.env)
nano /opt/themain/.env      # сохранить: Ctrl+O, Enter; выйти: Ctrl+X

# выйти из сервера
exit
```

---

## 7. Частые проблемы
- **`Connection timed out` / `refused`** — неверный IP или порт, либо сервер выключен /
  фаервол блокирует. Проверьте IP/порт в панели Beget; убедитесь, что VPS запущен.
- **`Permission denied (publickey,password)`** — неверный пароль или логин.
  Сбросьте root-пароль в панели Beget и попробуйте снова.
- **`Host key verification failed`** (после переустановки сервера) — на Mac/Linux выполните
  `ssh-keygen -R 85.198.xxx.xxx` и подключитесь заново.
- **Windows: `ssh` не распознан** — используйте PuTTY (вариант Б) или включите
  «OpenSSH Client» в Параметры → Приложения → Доп. компоненты.
